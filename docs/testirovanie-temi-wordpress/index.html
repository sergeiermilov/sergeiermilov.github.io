Методы тестирования темы WordPress

Попробуем разобраться в одном из актуальных вопросов — как протестировать новую тему WordPress на живом рабочем сайте.

Быстрый ответ

Самый простой способ — использование плагина WP Theme Test. Плагин какое-то время не обновлялся, так что сделайте бэкапы перед началом использования данного плагина на вашем сайте. Подробнее о данном методе мы поговорим ниже.

Вводная информация

Независимо от того, какую тему вы используете, возможность протестировать тему WordPress — одна из самых привлекательных вещей при создании нового сайта WordPress. Когда вы устанавливаете новую тему на исходный сайт, трудно понять, как она выглядит и функционирует. Есть несколько способов проверить новую тему, чтобы убедиться, что она работает точно, не подвергая опасности ваш живой сайт.

Когда вы выбираете тему, очень важно выбрать подходящую. Изучение отзывов или даже демонстрационная версия — это хорошее начало, но их недостаточно, чтобы судить о качестве темы. Например, конкретная тема может выглядеть впечатляюще, но иметь проблемы под капотом. В этой статье я расскажу о нескольких способах тестирования темы WordPress до и после ее покупки.

Почему важно протестировать тему перед использованием

Предположим, вы только что установили новую тему WordPress либо бесплатно, либо заплатив за нее огромную сумму. Определенно, следующее, что вы хотите сделать, это посмотреть, как она выглядит, и установить ее в качестве дизайна вашего сайта по умолчанию.

Итак, вы активируете тему, и тут вы понимаете, что она не подходит для последней версии WordPress. Она выглядит не так как надо и рушит весь дизайн вашего сайта. Ваши клиенты и посетители сайта начнут жаловаться на трудности при навигации по сайту. Вскоре вы вернетесь к старой теме оформления.

Это лишь пример ночных кошмаров, с которыми вы можете столкнуться при установке новой темы WordPress. Поэтому всегда полезно протестировать новую тему, прежде чем устанавливать ее в качестве дизайна сайта по умолчанию. Шаги ниже помогут вам определить, работает ли тема так, как вам нужно, совместима ли она с плагинами, которые вы планируете использовать, и как она обрабатывает различные медиа.

Существуют различные способы тестирования темы WordPress. Я расскажу вам о каждом из них шаг за шагом.

Метод 1. Тестирование темы с помощью локальной установки WordPress

Большинство из вас, возможно, не захотят тестировать тему прямо на своем сайте, потому что это может привести к серьезным последствиям. Поэтому лучшей альтернативой будет попробовать ее на сайте WordPress, который был установлен локально.

Есть темы, которые требуют соответствующей настройки для тестирования. Например, темы, созданные с помощью плагинов для формирования страниц, и темы для магазинов, использующие WooCommerce, требуют дополнительных плагинов для правильной работы. В таких случаях другие методы не подойдут, и вам придется использовать локальную установку WordPress для тестирования.

Локальная установка сайта WordPress позволит вам получить надежную тестовую среду для разработки сайта, поскольку локальный сайт WordPress не может быть доступен другим людям. Итак, как только у вас появилось место для тестирования, вы будете готовы приступить к работе!

Шаг 1. Скачиваем WordPress

Для начала скачайте последнюю версию WordPress — Theme Unit Test, которая позволит вам воспроизвести внешний вид сайта, существующего уже некоторое время. Как только вы импортируете все данные вашего сайта, WordPress естественным образом создаст кучу страниц, различных постов, пунктов меню и прочего. Это позволит вам провести гораздо более тщательное тестирование потенциальной темы.

Шаг 2. Импорт данных

Когда скачивание закончится, зайдите в панель управления вашего тестового сайта. Затем перейдите в «Инструменты -> Импорт« и найдите внизу опцию WordPress. Если вы используете этот инструмент впервые, вам нужно будет нажать «Установить сейчас«.

По окончании установки на этом месте появится ссылка «Запустить импорт», при нажатии на которую появится возможность загрузить XML-файл.

Нажмите кнопку «Выбрать файл», найдите на своем компьютере файл «WordPress Theme Unit Test Data», затем нажмите кнопку «Загрузить файл и импортировать».

На следующей странице WordPress предоставит возможность добавить вымышленных авторов для вашего фиктивного контента. Вы можете либо установить вымышленных авторов, либо оставить эти настройки без изменений, чтобы избежать сложностей.

В конце страницы вы также увидите опцию импорта вложений для вашего контента. Выберите ее, чтобы добавить изображения в посты и страницы, а затем нажмите на кнопку «Отправить». После этого на вашем сайте появится фиктивный контент, и вы сможете протестировать новую тему.

Шаг 3. Установите тему и начните тестирование

Таким образом, при локальной установке WordPress вы сможете протестировать тему с плагинами и даже добавить демо-контент для изменения дизайна без ущерба для вашего живого сайта.

Метод 2. Тестирование темы с помощью плагина

Если вы уже купили тему и скачали ее на свой компьютер, вы можете протестировать ее на своем сайте. Не волнуйтесь, данный метод не повредит вашему сайту. Он позволит вам протестировать и посмотреть тему, не меняя текущую тему вашего сайта.

WP Theme Test — это плагин, который позволяет протестировать другую тему на вашем сайте, не меняя тему по умолчанию и активную в данный момент. Таким образом, посетители вашего сайта будут видеть одну и ту же старую тему, в то время как администратор получает возможность тестировать новые темы.

Чтобы установить плагин, зайдите в панель управления WordPress и выберите «Плагины -> Добавить новый». Затем найдите «WP Theme Test» и нажмите «Установить» и «Активировать» для плагина из репозитория WordPress.

Следующим шагом будет установка темы, которую вы хотите протестировать. По окончании установки убедитесь, что тема пока не активирована!

Перейдите на страницу плагина WP Theme Test, нажав «Внешний вид -> WP Theme Test».

Здесь вы можете выбрать тему, которую хотите протестировать, с помощью выпадающего меню «Test Theme». Выберите тему, которую вы только что установили, и измените текущее состояние на ON. Не забудьте нажать «Сохранить изменения» (Save the changes).

Теперь откройте свой сайт, не выходя из админки WordPress, и вы увидите, как работает новая тема. Это означает, что новую тему могут увидеть только авторизованные пользователи и администраторы. Обычные посетители сайта по-прежнему будут видеть старую тему оформления.

Метод 3. Тестирование с помощью предварительного просмотра

Как уже говорилось, если тема выглядит аккуратно, это еще не значит, что она идеально подходит для вашего сайта или отлично функционирует. Поэтому, прежде чем нажать кнопку «установить/купить», чтобы получить выбранную вами тему WordPress, вы можете провести несколько тестов, чтобы убедиться, что она соответствует вашим требованиям.

Проверка совместимости

По сути, вам нужно убедиться, что тема, которую вы собираетесь использовать, совместима с новейшей версией WordPress, известными сторонними плагинами и различными браузерами.

Например, если вы планируете создать интернет-магазин, вам нужно купить тему, совместимую с плагином WooCommerce (или любым другим плагином для электронной коммерции), или тему, совместимую с конструктором страниц Elementor, если вы хотите плавно настраивать дизайн темы. Вы можете проанализировать совместимость темы, просмотрев страницу темы на торговой площадке.

Не забывайте про мобильные экраны

Также следует проверить, как тема выглядит на мобильных устройствах и в различных разрешениях. Удобный способ сделать это — использовать встроенные средства разработки вашего браузера для индивидуального тестирования темы на разных разрешениях устройств.

Для этого откройте демо-сайт темы, нажав на кнопку «Live Preview» на странице темы в маркетплейсе. Затем откройте консоль разработчика браузера, выбрав Control+Shift+I в Chrome (Control+Shift+M в Firefox). На этой странице вы можете выбрать различные настройки устройства для предварительного просмотра темы. Вот некоторые из основных тестов, которые вы можете использовать для проверки дизайна и отзывчивости темы перед покупкой.

После того как вы установите и активируете новую тему обычным способом, вы можете протестировать ее еще раз. Хотя это может занять некоторое время, процесс тестирования очень прост. Если вы решили воспользоваться этим способом, проверьте следующие пункты, чтобы убедиться, что ваша тема идеально подходит для сайта:

  • Зайдите на каждую из основных страниц вашего сайта и проверьте, правильно ли отображаются все элементы. Проверьте наугад несколько ваших постов, чтобы убедиться, что тема отображает их так, как вам нравится. Также убедитесь, что содержимое читаемо.
  • Проверьте виджеты, которые вы планируете использовать на своем основном сайте, и определите, правильно ли тема их отображает.
  • Установите все плагины, которые вам необходимы на вашем сайте, и проверьте, работает ли их функциональность с новой темой.

Если тема прошла все эти этапы, значит, вы готовы к работе. Как только вы будете готовы, вы сможете установить тему по умолчанию на свой новый сайт WordPress, а также внести дополнительные изменения в дизайн и оформление.

Бонусные инструменты для тестирования темы WordPress

Плагин Theme Check — это простой способ протестировать вашу тему и убедиться, что она соответствует последним стандартам проверки тем. С его помощью вы можете запустить все те же инструменты автоматического тестирования вашей темы, которые WordPress.org использует для проверки тем.

Тесты запускаются через простое меню администратора, и все результаты отображаются сразу. Плагин поддерживается собственными разработчиками WordPress, и его использование — самый простой способ убедиться, что ваша тема на 100% соответствует стандартам WordPress.

InstaWP — это онлайн-песочница WordPress, которая может послужить идеальным испытательным полигоном для ваших разработок. Эта среда позволяет создавать сайты WordPress для тестирования менее чем за секунду, имеет SSL-сертификаты, автоматический домен, возможность настраивать версии PHP и WP, а также поставляется со встроенными инструментами.

У них также есть расширение для браузера Chrome, которое выступает в качестве вспомогательного приложения для InstaWP. Используя это расширение, вы можете запускать экземпляры WordPress из репозитория. После установки зайдите в любую тему и нажмите на кнопку «Запустить». Вы можете протестировать тему, не скачивая и не устанавливая ее. Функция «песочницы» бесплатна, но у них также есть премиум-планы, которые начинаются от 9 долларов в месяц, что открывает еще больше возможностей.

TasteWP — это новая платформа, служащая местом разработки и тестирования, где разработчики могут экспериментировать с основным WordPress, плагинами, темами, виджетами и другими дополнениями для WordPress. Эта «песочница» WordPress позволяет пользователям запустить новый экземпляр WordPress в считанные секунды. Это проще, чем поддерживать локальную среду разработки, на создание которой многие случайные пользователи WordPress не тратят время.

Бесплатный генератор документации наверняка станет вашим хорошим другом в работе с документацией по вашей теме. Помимо помощи в написании документации, он генерирует красивый формат без дополнительной работы с вашей стороны. Это отличная помощь, особенно для авторов маркетплейсов, которые очень строго относятся к документации.

WP Test имеет просто фантастически исчерпывающий набор тестовых данных для проверки функциональности ваших тем. Многие разработчики, особенно разработчики распространяемых тем, используют WP Test при создании программ, ориентированных на WordPress. Данные WP Test основаны на рекомендациях из WordPress’s Theme Unit Test, найденных в кодексе. Сайт очень прост и включает в себя только кнопку загрузки и демонстрации.

Theme Unit Test — это ручные тесты, с помощью которых можно проверить функциональность темы и ее реакцию на нестандартные ситуации с контентом и настройками.

Созданные командой проверки тем WordPress, модульные тесты тем помогут убедиться, что ваша тема работает идеально в любых условиях. Это файл экспорта WordPress, который можно импортировать в любую установку WordPress с помощью WordPress Importer. Вы можете импортировать его в локальную среду разработки, чтобы протестировать свою тему.

Заключение

Это некоторые из удобных для новичков методов проверки темы WordPress. В целом, всегда лучше держаться подальше от устаревших и пиратских тем WordPress, чтобы обеспечить безопасность вашего сайта.

И старайтесь покупать темы WordPress из надежных источников, которые продают темы, созданные с использованием новейшего кода и технологий. Если вы выбираете бесплатные темы, всегда скачивайте их из репозитория WordPress. Надеюсь, эта статья помогла вам узнать, как проверить тему WordPress на соответствие последним стандартам и как её протестировать на живом сайте.
